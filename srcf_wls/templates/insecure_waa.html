Insecure web application: login blocked

The web application at {{ domain }} wants you to log in, but is not communicating securely.

To avoid leaking your identity to eavesdroppers, we have configured the authentication service to forbid logging into insecure web applications.

What does this mean?

Websites that talk to your browser using encryption will use addresses that start with https. The web application at {{ domain }} that sent you here asked that we send the result of your login to an address starting with {{ scheme }}, which does not provide encrypted communications.

Anyone who can snoop on your connection, for example when you are connected to an unsecured public network, could capture and store the responses that are sent by this service to web applications using insecure {{ scheme }} addresses.

At best, an attacker with a capture of a valid authentication response would just find out your username. At worst, they could "replay" the same response to the same web application, and log in as you.

Whether a replay attack like this would be possible can depend on many factors, including how robust and strict the target web application is in its checking of authentication responses, but it is safest to make these attacks impossible in the first place.
