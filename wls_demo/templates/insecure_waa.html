Insecure web application

The web application at {{ domain }} requesting your identity is not secure.

To avoid leaking your identity to eavesdroppers, we have configured this login service not to respond to insecure web applications.

What does this mean?

Websites that communicate in an encrypted manner with you use addresses that start with https. The web application that sent you here requested that we send the result of your login to an address starting with {{ scheme }}, which does not provide encrypted communications.

Anyone who is able to snoop on your connection, for example when you are connected to an unsecured public network, can capture the authentication responses sent by this service to web applications using insecure {{ scheme }} addresses.

At best, an attacker with a capture of a valid authentication response would just find out your username. At worst, they could successfully "replay" the same response to the same web application to try to log in as you. Whether a replay attack like this would be possible can depend on many factors, including how robust and strict the target web application is in its checking of authentication responses.
